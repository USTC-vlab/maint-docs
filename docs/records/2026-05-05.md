## 迁移 `/opt/vlab`

因为 pv1 没有容器在使用 `/opt/vlab`，所以提前在 pv1 上做好准备工作，把 `/opt/vlab` 格式化好，并从 pv2 拉回数据：

```shell
umount /opt/vlab
mkfs.btrfs -f /dev/pve/vlab-software
vim /etc/fstab
# /dev/pve/vlab-software /opt/vlab btrfs rw,noatime,compress=zstd:3 0 1
systemctl daemon-reload
mount /opt/vlab
rsync -aHAXxv pv2:/opt/vlab/ /opt/vlab/
# dedup?
btrfs subvolume snapshot /opt/vlab /opt/vlab/20260505-send
```

接下来对于所有其他机器，按上述操作执行到 `rsync` 步骤之前，然后用 `btrfs send` 代替 `rsync`：

```shell
# 目标机器上
nc -l -p 12345 | btrfs receive /opt/vlab

# pv1 上
btrfs send --compressed-data /opt/vlab | pv > /dev/tcp/$target/12345
```

发送完成后，目标机器上的内容位于 `/opt/vlab/20260505-send`，需要把它重新创建成名为 `@` 的子卷并设为默认：

```shell
btrfs subvolume snapshot /opt/vlab/20260505-send /opt/vlab/@
btrfs subvolume delete /opt/vlab/20260505-send
btrfs subvolume set-default /opt/vlab/@
umount /opt/vlab
mount /opt/vlab
```

最后对于 pv1 自己，将 `/opt/vlab` 重新创建成子卷并设为默认：

```shell
btrfs subvolume snapshot /opt/vlab/20260505-send /opt/vlab/@
btrfs subvolume delete /opt/vlab/20260505-send
btrfs subvolume set-default /opt/vlab/@
# 删除 @ 以外的内容
umount /opt/vlab
mount /opt/vlab
```

## 迁移 `proxmox-boot-tool`

`proxmox-boot-tool` 由 `proxmox-ve` 依赖，因此无需另行安装。

首先 `lsblk` 找到所有 boot 分区（512M 或 1G 大小的 EFI 分区），全部 `umount` 掉，然后对**每个**分区执行：

```shell
proxmox-boot-tool format /dev/sdXX --force
proxmox-boot-tool init /dev/sdXX grub
```

重启验证机器能正常启动即可。
